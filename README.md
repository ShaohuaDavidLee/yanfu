# 严复 Skill：你的产品故事翻译官
<img width="1638" height="687" alt="image" src="https://github.com/user-attachments/assets/a14785cf-7ef6-4749-9e95-c6c181cfe9a5" />


把你的“开发者视角”落地页，翻译成用户 5 秒能懂的落地页。

严复不负责重新定位产品，也不替你发明价值主张。它只做一件事：忠于你已经做出来的产品和现有页面，把功能、机制、截图和 CTA 重新组织成用户能听懂、记住、复述的故事。

## 一句话

严复把开发者写给自己的 Landing Page，翻译成用户一眼知道“这是什么、为谁做、能帮我得到什么”的 Landing Page。

## 适合谁

- 已经有一个产品、Demo、MVP 或上线页面的开发者。
- 页面能说明产品功能，但用户看完不知道为什么重要。
- 不想先改定位、改产品、重做品牌，只想先把故事讲清楚。
- 想把 Go To Market 的第一步收缩成一个能被测试的落地页。

## 不适合什么

- 只给一个模糊 idea，让严复帮你判断值不值得做。
- 要求严复做竞品分析、产品诊断、商业模式判断。
- 希望严复重做品牌、重画 UI、改目标用户、发明新承诺。
- 只有登录后页面，又不愿提供截图、录屏或介绍材料。

这些工作更接近包青天、产品顾问或设计重做，不是严复 v0.1 的职责。

## 输入

至少提供：

1. 一个公开可访问的 Landing Page URL。
2. 1-5 张核心产品界面截图。

可选提供：

- 你最不希望被改掉的一句话。
- 真实 CTA 和链接。
- Logo、字体、品牌素材。
- 不能删除或不能改写的内容。

注意：如果你在严复官网上传截图，复制提示词时图片不会自动进入剪贴板文本。你需要在自己的 agent 对话里手动把截图作为附件一起发过去。

## 输出

严复交付两份文件：

- `landing.html`：一个可直接打开、响应式的落地页。
- `yanfu-notes.md`：严复译注，解释哪些内容被保留、压缩、重排，以及哪些内容需要你确认。

`yanfu-notes.md` 必须包含：

- `信`：哪些表达忠于原页面与截图证据。
- `达`：一句话 Pitch、故事主线和信息顺序。
- `电梯演讲`：一个 30 秒版本，帮助你向朋友、用户或投资人快速说明产品。
- `雅`：保留了哪些原设计元素，只做了哪些克制调整。
- `待确认`：哪些承诺或事实证据不足，暂时不能写进页面。

## 严复三原则

### 信

只忠于原始材料：公开 Landing Page、产品截图、开发者明确补充的信息。没有证据的数据、评价、客户案例、节省时间、准确率、增长效果，都不能写进页面。

### 达

把“功能是什么”翻译成“用户能做什么”和“用户得到什么”。页面首屏必须让陌生用户快速回答：这是什么产品？为谁服务？解决什么问题？下一步该做什么？

### 雅

结构清楚，表达克制。严复可以重排故事、压缩文案、调整信息层级和移动端可读性，但不要重新设计。默认忠于原 Landing Page 的设计风格：品牌气质、主色、字体气质、组件语言、视觉密度和真实 CTA 都应保留。

## 给用户的使用提示词

```text
你是严复，产品故事翻译官。请把下面这个“开发者视角”的落地页，翻译成用户 5 秒能懂的落地页。

严复 Skill 安装地址：https://github.com/ShaohuaDavidLee/yanfu

原文链接：（粘贴你的 Landing Page URL）
产品截图：（手动附加 1-5 张核心产品界面截图；截图无法进入剪贴板文本）

硬性边界：不要重新设计。请忠于原 Landing Page 的设计风格，保留原品牌气质、主色、字体气质、组件语言和视觉密度；暂不把产品截图当作视觉设计来源，截图只用于理解产品事实与界面证据。

翻译三则：
一、信——只忠于以上材料里的事实，不重定位、不夸大、不替产品发明价值；
二、达——让陌生用户 5 秒内明白产品是什么、为谁服务、解决什么；
三、雅——结构清楚、表达克制，并能一句话传播。

请输出 landing.html 与 yanfu-notes.md。yanfu-notes.md 中请包含：信、达、电梯演讲、雅、待确认。
```

## 给 agent 的执行约束

当你作为 agent 使用本仓库时：

- 先读取 `skills/yanfu/SKILL.md`。
- 再按需读取 `skills/yanfu/references/evidence-and-browsing.md`、`translation-method.md`、`output-spec.md`。
- 不得自行注册或登录产品。
- 不得重新定位产品、修改目标用户、发明承诺或进行竞品分析。
- 不要重新设计；忠于原 Landing Page 的设计风格。
- 生成后运行 `skills/yanfu/scripts/validate_delivery.py` 校验交付目录。

## 仓库结构

- `skills/yanfu/`：严复 Skill 源文件、参考说明与交付校验脚本。
- `landing/index.html`：严复官网首页，可作为静态站点部署。
- `landing/assets/bundle/`：官网页面的图片、字体与运行时资源。
- `tests/`：Skill 与官网页面的契约测试。
